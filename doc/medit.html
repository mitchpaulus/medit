medit

A small modal text editor. Vim-like motions, Kakoune-like selection-first editing, multi-cursor.

What every key does, by modifier. Letters/symbols show the glyph; the caption is the action. Dimmed keys are unbound. Lower- and upper-case maps are normal mode; the Ctrl/Alt maps fold in insert-mode (readline) bindings, colour-coded by mode.

KeyAction

Normal mode — edit

KeyAction

I / AInsert at line start / line end
o / OOpen line below / above

Ex commands (:)

CommandAction
:wWrite current buffer (refuses if file changed on disk)
:w!Force write, overwriting external changes
:qClose active buffer (prompts if unsaved); exits the editor when the last buffer is closed
:q!Force close active buffer, discarding unsaved changes
:wqWrite active buffer and close it
:e <path>Open file (switches to it if already open)
:e!Reload current buffer from disk, discarding in-memory edits
:bn / :bnextNext buffer
:bp / :bprev / :bpreviousPrevious buffer
:ls / :buffersList open buffers

Dirty buffers show [+] in the status line. q in normal mode and :q close the active buffer (prompting if it's unsaved); the editor exits when the last buffer is closed. Ctrl+c exits the whole editor; if any buffer is unsaved you're prompted — y saves all and quits, n discards and quits, anything else cancels.

Files are watched for external changes via inotify (Linux), ReadDirectoryChangesW (Windows), or FSEvents (macOS). If a buffer is clean and the file changes on disk, medit auto-reloads silently. If the buffer is dirty, you'll see [disk!] next to the filename and a hint; resolve with :e! to take the disk version (losing your edits) or :w! to overwrite the disk version (losing the external edits).
